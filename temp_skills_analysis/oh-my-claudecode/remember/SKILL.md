# Remember

## 描述

当用户想要保存或组织会话中发现的有用知识时，使用此技能。

目标：将持久的、可复用的知识提升到正确的记忆表面，而不是让它埋没在聊天历史中。

## 使用场景

- 用户说 "remember this"、"save this"、"note this"、"don't forget"
- 会话中发现了值得持久化的项目知识
- 需要整理或清理已有的记忆表面
- 用户想要将临时笔记升级为持久知识

## 指令

### 记忆表面

- **项目记忆** — 持久的团队/项目知识
- **笔记本优先** — 下几轮的短期高信号上下文
- **笔记本工作** — 临时活跃会话笔记
- **文档 / 配置文件** — 真正属于那里的持久指令和约定

### 工作流

1. 收集相关的会话发现
2. 对每个项目分类：
   - 持久的项目事实
   - 临时工作笔记
   - 操作者偏好或指令
   - 重复 / 过时 / 冲突的信息
3. 为每个项目提议最佳目的地
4. 仅写入或更新适当的记忆表面
5. 指出应清理的重复或冲突

### 规则

- 不要将所有内容倾倒到一个存储中
- 优先使用项目记忆存储持久团队知识
- 优先使用笔记本存储短期工作上下文
- 保持条目简洁和可操作
- 如果某事不确定，标记为不确定而非作为事实存储

## 示例

**输出格式：**
- 存储了什么
- 存储在哪里
- 发现的任何重复/冲突
